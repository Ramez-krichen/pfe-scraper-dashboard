# Competitive Intelligence Automation System

A full-stack, AI-powered system for monitoring and analyzing competitor ecommerce sites. This system coordinates a web scraper, an n8n automation workflow, and a Next.js dashboard to provide strategic business insights.

## 🏗️ Project Structure
- **/scraper-main**: FastAPI API and Playwright-based background worker.
- **/interface/dashboard**: Next.js 14 Dashboard.
- **/interface/schema**: Database initialization scripts.

---

## 🚀 How to Run Everything (Step-by-Step)

### 1. Prerequisites
- **Docker Desktop**: To run Redis, Postgres, and the Scraper services.
- **Node.js 20+**: To run the Dashboard locally.
- **n8n (Optional/External)**: For AI orchestration nodes.

### 2. Launch the Infrastructure (Docker)
Ensure Docker Desktop is running, then execute the following command at the **project root**:
```bash
docker-compose up -d --build
```
This will start the following services:
- **Redis**: Port 6379 (Used by the Scraper for job queuing).
- **PostgreSQL**: Port 5432 (Automatically initialized with the `init.sql` schema).
- **Scraper API**: Port 8000 (Ingestion service for URLs).
- **Scraper Worker**: Background crawler instance.

> [!NOTE]
> The initial build can take **30-40+ minutes** as it pulls large browser binaries for Playwright.

### 3. Verify System Health
Check if the services are running:
```bash
docker-compose ps
```
Verify the Scraper API is alive:
```bash
curl http://localhost:8000/health
```

### 4. Start the Dashboard (Next.js)
Open a new terminal and navigate to the dashboard directory:
```bash
cd interface/dashboard
npm install
npm run dev
```
The UI will be available at [http://localhost:3000](http://localhost:3000).

## 🧪 Testing the Scraper
To manually add a URL to be crawled, run the following CURL command:
```bash
curl -X POST http://localhost:8000/frontier/urls \
     -H "Content-Type: application/json" \
     -d '{"url": "https://example.com", "priority": 5}'
```

## ⚠️ Configuration Notes
- **Webhooks**: Inside Docker, the scraper attempts to reach n8n via `http://host.docker.internal:5678`.
- **Database**: The dashboard connects to `localhost:5432` via credentials defined in `interface/dashboard/.env.local`.
